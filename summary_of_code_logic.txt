陈诗薇 - git bash推送方法：

配置 Git 身份（只做 1 次）
打开 Git Bash，输入下面两句命令，换成自己的 GitHub 信息：
bash
运行
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"
只需要配置一次。
二、创建本地项目与文件
1. 新建项目文件夹
在电脑上随便找个地方（比如 E 盘），新建一个文件夹，比如叫 云台和gitbush，把你的代码 / 文档都放在里面。
2. 创建要推送的文件
要推送 summary_of_code_logic.txt，直接在文件夹里新建文本文档，写好内容保存就行。
三、GitHub 端操作（只做 1 次）
1. 新建空仓库
登录 GitHub，点击右上角 + → New repository
仓库名随便起（比如 gimbal-project-txt），一定要把所有勾选框都取消（不要勾 README、License 这些），点 Create repository
复制仓库的 HTTPS 地址，比如：https://github.com/你的用户名/仓库名.git
四、第一次推送完整流程（从 0 上传文件）
1. 进入项目文件夹
打开 Git Bash，用 cd 命令进入你的项目文件夹（路径有空格要加引号）：
bash
运行
cd "/e/云台和gitbush"
回车后，Git Bash 路径会变成 /e/云台和gitbush。
2. 初始化本地 Git 仓库
bash
运行
git init
这一步会把你的文件夹变成 Git 能管理的仓库，只做一次。
3. 添加文件到暂存区
bash
运行
git add .
. 代表 “当前文件夹下所有文件”，会把你写的 summary_of_code_logic.txt 都加进去。
4. 提交文件到本地仓库
bash
运行
git commit -m "第一次提交：上传云台逻辑与Git推送说明文档"
-m 后面的内容要写清楚，方便以后回溯版本。
5. 关联 GitHub 远程仓库
把你之前复制的仓库地址粘进去：
bash
运行
git remote add origin https://github.com/你的用户名/仓库名.git
6. 推送到 GitHub
bash
运行
git push -u origin master
（注意：你本地的默认分支是 master，不是 main，所以用 master）
五、日常修改后更新推送（每次修改都用这 3 步）
以后修改了文件（比如更新了文档内容），只需要执行这 3 条命令：
bash
运行
# 1. 把修改过的文件加入暂存区
git add .

# 2. 提交修改，写清楚这次改了什么
git commit -m "更新：补充云台控制的PID参数说明"

# 3. 推送到 GitHub
git push





陈诗薇--云台逻辑：
一．	云台是机器人上控制枪口（发射机构朝向）的结构 
主要有两个方向：yaw:左右转
                pitch上下抬
电控的目标是根据让yaw和pitch按照目标稳定运行，同时通过传感器反馈让云台不乱转、不抖动、不撞机械限位
二．	
云台主任务逻辑：
1. 设置云台模式，
2. 模式切换过渡
3. 更新反馈
4. 设置目标
5. PID 计算 
6. 发射机构控制
7. CAN 发送

三、云台模式逻辑
- GIMBAL_ZERO_FORCE：无力模式。
- GIMBAL_INIT：初始化模式
- GIMBAL_CALI：校准模式。
- GIMBAL_MOTIONLESS：保持不动。云台不继续响应输入，尽量保持当前状态。
- GIMBAL_RELATIVE_ANGLE：正常跟随模式。遥控器摇杆和鼠标控制 yaw、pitch 目标角度。
- GIMBAL_SPIN：小陀螺模式。底盘旋转时，云台仍然可以配合遥控器或鼠标进行控制。
    
四，云台控制（负责真正算电机输出）：目标角度是多少，当前角度是多少，误差多少，电机应该给多少电流
两者关系：云台主任务
          ↓ 调用
云台控制函数
           ↓ 计算
电机电流
  ↓
CAN 发送
五、遥控器输入逻辑

遥控器摇杆控制的是目标角度变化，不是直接控制电机。
六、传感器反馈逻辑
   主要反馈有：
- 编码器：知道电机相对于机械中位转了多少（相对角）。
- 陀螺仪：知道云台当前转得多快。
- INS 姿态解算：知道云台当前 yaw、pitch 姿态角。
七、PID 控制逻辑
PID 的作用是根据目标值和当前值的误差，计算电机应该输出多大电流。
